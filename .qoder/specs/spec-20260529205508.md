# UI 美化实施计划 — 方案三：混合方案

## Context

当前博客项目的 UI 存在以下问题：
- **整体风格朴素**：背景平淡、卡片缺乏视觉层次、导航栏过于简单
- **组件样式单调**：按钮、标签、卡片仅依靠 1px 边框区分，缺乏阴影和层次
- **视觉层次单一**：标题/正文/标签没有明显大小对比，阅读体验一般

用户希望采用 **玻璃态 (Glassmorphism)** 设计风格，同时参考 **Medium** 的简洁阅读体验。

**解决方案**：采用混合方案 —— 阅读区保持 Medium 风格的纯净排版，交互区（Header、侧边栏、卡片）使用玻璃态增加精致感和视觉层次。

---

## 实施步骤

### 第 1 步：全局样式增强 (`global.css`)

**文件路径**: `frontend/src/styles/global.css`

**改动内容**：
1. **阅读区排版增强**：
   - 增加 `.reading-area` 工具类（max-width: 680px，居中，左右 padding）
   - 优化 `.article-body` 样式：字号增大到 18px，行高 1.85，标题层次对比增强
   - 优化 blockquote 样式：左侧 3px amber 边框，渐变背景，圆角
   - 优化代码块样式：圆角 12px，微妙阴影，适当外边距

2. **玻璃态工具类**：
   - 添加 `.glass-panel` 类：`bg-white/75 backdrop-blur-xl border border-white/50`（浅色模式）
   - 添加 `.glass-panel-dark` 类：`dark:bg-stone-900/65 dark:backdrop-blur-xl dark:border-white/8`
   - 添加 `.glass-card` 类：带阴影的玻璃卡片

3. **修复已知问题**：
   - 将 `#root { padding-top: 73px }` 改为动态计算或使用 sticky Header
   - 统一容器最大宽度策略

### 第 2 步：导航栏玻璃化 (`Header.jsx`)

**文件路径**: `frontend/src/components/layout/Header.jsx`

**改动内容**：
1. **背景玻璃效果**：
   ```jsx
   // 当前
   bg-stone-50 dark:bg-stone-950 py-6
   
   // 改为
   bg-stone-50/80 dark:bg-stone-950/70 backdrop-blur-xl py-4
   border-b border-stone-900/[0.06] dark:border-white/[0.06]
   ```

2. **导航链接指示器**（Medium 风格下划线）：
   ```jsx
   // 激活链接
   after:absolute after:bottom-0 after:left-0 after:right-0
   after:h-[2px] after:bg-amber-500 after:rounded-full
   after:transition-transform after:duration-300 after:origin-center
   after:scale-x-100
   
   // 非激活链接
   after:scale-x-0 hover:after:scale-x-100
   ```

3. **Logo 品牌强化**：
   ```jsx
   <Link to="/" className="text-xl font-bold text-stone-900 dark:text-stone-100 tracking-tight">
     Code<span className="text-amber-600 dark:text-amber-400">Ink</span>
   </Link>
   ```

4. **搜索框优化**：
   - 宽度从 `w-40` 增加到 `w-48`，改善中文输入体验
   - 添加玻璃态背景：`bg-white/60 dark:bg-stone-900/50 backdrop-blur-sm`
   - 聚焦时增加光晕效果：`focus:ring-2 focus:ring-amber-500/10`

## 验证方案

### 1. 视觉验证
- 浅色模式下检查所有页面的玻璃效果是否正常显示
- 深色模式下检查对比度和边框可见性
- 移动端（375px 宽度）检查响应式布局
- 宽屏（1920px 宽度）检查三栏布局

### 2. 功能验证
- 页面切换动画是否流畅
- 导航链接指示器是否正确显示
- 搜索框聚焦效果
- 按钮 hover/click 效果
- 卡片 hover 上浮效果

### 3. 性能验证
- `backdrop-blur` 在低端设备上是否有明显卡顿
- 页面加载时是否有布局抖动

### 4. 浏览器兼容性
- Chrome/Edge：完整支持
- Firefox：完整支持
- Safari：需要 `-webkit-backdrop-filter` 前缀（Tailwind 自动处理）

### 5. 启动验证
```bash
cd frontend
npm run dev
# 浏览器访问 http://localhost:5173
# 逐页检查效果
```

## 注意事项

1. **保留当前配色体系**：Stone + Amber 不变，仅优化饱和度和层次
2. **渐进增强**：即使 `backdrop-blur` 不生效，页面仍可正常阅读
3. **移动端简化**：在小屏幕下可以适当减弱玻璃效果以提升性能
4. **保持一致性**：所有同类组件（卡片、按钮、标签）使用统一的样式规范
